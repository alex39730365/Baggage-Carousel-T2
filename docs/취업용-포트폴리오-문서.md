# 취업용 포트폴리오 문서 — 공항 수하물 캐러셀 현황 대시보드

이 문서는 **이력서·자기소개서·포트폴리오 사이트**에 옮겨 적기 쉽도록 프로젝트를 한국어로 정리한 것입니다. 앱 UI는 그대로 두고, 설명만 문서로 관리합니다.

---

## 1. 프로젝트 한 줄 소개

공공데이터 API로 도착편·수하물 캐러셀 정보를 가져와, 엑셀 현황표와 유사한 **시간 × 캐러셀** 격자와 목록 형태로 보여 주는 **React + TypeScript** 웹 대시보드입니다.

---

## 2. 기술 스택

| 구분 | 사용 기술 |
|------|------------|
| UI | React 18, TypeScript, Tailwind CSS |
| 빌드 | Vite |
| 데이터 | 공공데이터포털 오픈 API (도착편·수하물 캐러셀 관련) |
| 배포(선택) | Vercel, 서버리스 함수로 API 중계 (`api/baggage-arrivals.ts`) |

---

## 3. 구현 내용 (면접·서류용 bullet)

- API 응답 `item[]`을 **시간(`HH:00`)·캐러셀 번호** 기준으로 정규화해 격자 셀에 매핑
- 필드명이 환경마다 달라질 수 있어 **`flightId`, `estimatedDatetime`, `lateralNo` 등** 복수 키를 순차 탐색하는 파싱 로직
- **주기적 재호출·로컬 캐시·고정 스케줄 병합**으로 데이터 공백을 줄이는 흐름 설계
- 터미널 탭, 카드/테이블 뷰 전환, 모바일에서 격자 확대, 편명 검색·하이라이트·로컬 저장 등 **실사용에 가까운 UX** 구성
- 배포 시 **서버리스로 공공 API를 프록시**해 클라이언트에 인증키를 노출하지 않는 구조(권장)

---

## 4. 프로젝트 구조 요약

- `src/components/BaggageCarouselBoard.tsx` — 격자·목록·필터·검색 등 메인 화면
- `src/hooks/useBaggageData.ts` — 데이터 fetch, 캐시, 갱신 주기
- `src/lib/baggageApi.ts` — API 호출·슬롯 정규화·정렬·중복 처리
- `src/types.ts` — 도메인 타입
- `api/baggage-arrivals.ts` — Vercel용 공공 API 중계

---

**프로젝트명:** 공항 수하물 캐러셀 현황 대시보드

공공데이터 API 연동 React(TypeScript) 웹앱. 도착편·캐러셀 정보를 시간×캐러셀 격자로 시각화하고, 반응형 UI·상태 캐시·Vercel 서버리스 API 프록시를 구현.

---

## 6. 자기소개서용 문단 (예시, 본인 말투로 수정)

이 프로젝트에서는 공항 수하물 캐러셀 공공 API를 활용해, 운영 현장에서 쓰는 현황표와 비슷한 형태로 정보를 보여 주는 대시보드를 구현했습니다. 단순 표 출력에 그치지 않고, API 필드가 환경마다 달라질 수 있는 점을 고려해 여러 키를 순차적으로 읽는 방식으로 안정성을 높였고, 주기적 갱신과 로컬 캐시를 조합해 사용자가 끊김 없이 화면을 볼 수 있도록 했습니다. 또한 모바일 터치 환경을 고려한 레이아웃과 검색·강조 기능을 넣어 실제 사용 시나리오를 염두에 두었습니다. 배포 환경에서는 서버리스 함수로 API를 중계해 키 관리와 CORS 이슈를 줄이는 방향으로 마무리했습니다.
